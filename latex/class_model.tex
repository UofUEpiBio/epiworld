\doxysubsection*{Setting and accessing parameters from the model}
\label{_amgrp61207d51bcc1b661804b11cec5dc845e}%
 \mbox{\hyperlink{class_tools}{Tools}} can incorporate parameters included in the model. Internally, parameters in the tool are stored as pointers to an std\+::map$<$$>$ of parameters in the model. Using the {\ttfamily epiworld\+\_\+fast\+\_\+uint} method directly fetches the parameters in the order these were added to the tool. Accessing parameters via the {\ttfamily std\+::string} method involves searching the parameter directly in the std\+::map$<$$>$ member of the model (so it is not recommended.)

The {\ttfamily par()} function members are aliases for {\ttfamily get\+\_\+param()}.

In the case of the function {\ttfamily read\+\_\+params}, users can pass a file listing parameters to be included in the model. Each line in the file should have the following structure\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{[name of parameter 1]: [value in double]}
\DoxyCodeLine{[name of parameter 2]: [value in double]}
\DoxyCodeLine{...}

\end{DoxyCode}


The only condition for parameter names is that these do not include a colon.
